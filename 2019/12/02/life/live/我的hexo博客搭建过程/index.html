我的hexo博客搭建过程

1.为什么我要选择搭建一个博客系统？

1.大老师说：我乐意，哈哈因为我需要！
  羡慕别人的博客网站，自己为什么就不能搭一个。
  博客系统看着就香！

2.自己的博客系统自己做主，主要是买了台阿里服务器和域名，总得用下吧！

3.急需一个博客网站来记录我的思想。每日一总结 + 综艺大集合 + 电影大集合
+ 游戏大集合 + hiphop，太想拥有一个属于自己的世界。

4.编程资料太多需要整理归纳，然后方便以后开发查阅

5.一些自己开发工具整理下，以后还访问啥其他导航栏，自己访问自己博客点链接他不香吗？

诸如此类的小思想.....

现在学搭博客一时爽，一直搭一直爽

2.搭博客前的故事

博客系统说白了就是一个网站，有前端有后端有数据，系统架构怎么实现呢，
刚开始肯定以为很复杂，想着是不是要部署一个nginx+tomcat+mysql这样的一套系统上去，一想到前端直接就放弃了，这也是最开始对待博客系统的看法。

然后，想着自己很多demo工程需要用到linux，老是用vmware总感觉不舒服，索性自己买一个玩一下，也不贵。再就看到了阿里云上面的服务器有定制化博客系统，但是价格贵了些，我又是第一次尝试觉得用别人搭好的，自己好逮是个程序员，觉得丢人，开始自己刚一波。

开始了，第一天按照自己的思路装一天我们熟悉的环境直接干蒙，一是好久没使用linux，二是软件安装步骤好多，有些许的绝望，还好中间遇到了docker这个神器，我直接忘记了我需要搭博客了，转头直接去学docker了,本来以前是有教的，自己没好好学，用的时候才发现TM是个神器。

偶然间，上了下B站，然后我就尴尬了，遇到了hexo这个博客框架，看到视频大佬如此顺滑，我TM还等啥，干啊！

3.搭建博客

bilibili:https://www.bilibili.com/video/BV1Yb411a7ty?from=search&seid=8571377819040181467

对应博客：http://www.dragonbaby308.com/hexo/

这位大佬的博客下的很多工具，真的是很舒服，我以后直接盗过来用了，看着就香！

跟着他的步骤，很快就在服务器上搭起来了，中间出了个错好像是更换主题出的错
Hexo启动页面显示extends includes/layout.pug block content include includes/recent-posts.pug include

解决办法是：https://blog.csdn.net/weixin_44318830/article/details/104884936

再次搜索知乎上也有类似解决方案。

后面开始去给我的博客换主题了，去知乎看了下，找到了butterfly主题，发现很符合我的要求，然后找了好几个，主题很容易就换成功了
主要的还是参照butterfly本人的配置来优化博客主题。

https://demo.jerryc.me/posts/dc584b87/

这里因为自己急于求成，直接去引用别人完整的_congig.yml配置导致出现了一堆问题，直接把我给干懵了。
刚开始接触这个觉得太多很好的功能了，自己上去就想都配置好，搞得我一点想法都没有。

这里域名绑定github，我中途没绑定对，我直接去绑定了服务器，每次启动都要走端口，然后部署到github上的博客因为自己用户名和工程名不一致，导致我直接xxx.github.io访问无效。坑爹的是这样不对导致，域名绑定不上去，
域名绑定我又出了两个问题：第一个就是上面这个问题，github工程名和用户名不一致，域名绑定出问题，第二个就是CNAME文件我放错位置了，这就导致每次我都需要重新去github上面绑定。
正确的打开方式是：
https://www.jianshu.com/p/a1a4ebf0e5fc

这里我没有绑定Coding，所以搞得域名只能走外网访问。

6.博客搭建过程出的错

1.can not read a block mapping entry; a multiline key may not be an implicit key at line 161, column 6:\n    cover:\n         ^' 
   _config.yml文件中多出无效字母，无效标签
   很可能是手误导致，报错信息如上

2.域名绑定不稳定问题，每次都需要重新去github的setting里面去绑定
   原因是：CNAME文件并没有放在正确的位置并且提交到git上

3.痛点：每次hexo d都需要输入github的用户名和密码？
  参考文章：https://www.dazhuanlan.com/2019/11/06/5dc220d9811d2/
  具体解决方案在最后
      ssh -T git@github.com
repo: git@github.com:username/username.github.io


4.原来我的categories多了一些---，是因为我二次添加的，原因是typora不显示，导致报错，
参考文章：https://github.com/hexojs/hexo/issues/2717
解决方案是：去除多余部分
